Error_1:

UnicodeDecodeError: 'ascii' codec can't decode byte 0xe5 in position 1: ordinal not in range(128)

出现这个问题的原因主要是python环境中的编码问题，ascii和unicode混淆，可以通过以下代码解决问题

decode或者encode的时候，如果不指定编码方式，会按照缺省编码方式来处理。查看、设置缺省编码方式可以这样做：

import sys
reload(sys)
sys.setdefaultcoding('utf8')#将默认环境编码方式设置为utf8

encode()是将string转化为unicode

decode()是将unicode转化为string

unicode()函数其实是调用string的decode方法

unicode可以作为中间码，当print一个unicode时，多数可以正常输出对应字符

Error_2:

IndentationError: unindent does not match any outer indentation level

当同一段代码的缩进中，出现既有4个空格还有<tab>键，就会报上面的错

Error_3

expected a string or other character buffer object

出现上面的错误时，比如：

floor = 'strstrstr',str(123),'strstrstr'
with open('Error.txt') as f:
    f.write(floor)

上面的floor变量应该改为floor = 'strstrstr'+str(123)+'strstrstr ‘

Error_4

TypeError: cannot concatenate 'str' and 'int' objects

在将str和int两种类型的数据进行结合时出现的错误，解决方法：

  1. print ‘Is your secret number “ + str(p) + “?”
  2. print ‘Is your secret number %d?”%p
    (for multiple integers, you can ‘%d %d %d’%(num1, num2, num3)
  3. print ‘Is your secret number {0}?”.format(p)

Error_5

<urlopen error [Errno 10060] >

当使用urllib2的urlopen()函数，打开链接时，可能出现这种错误，有可能是因为 主机所在网络不稳定，换个网络环境，或者等一段时间再行测试。
