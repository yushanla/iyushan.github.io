windows_server下shadowsocks配置

之前已经配过一遍这个了，当时也已经配完可用，这几天服务器关机了，服务器重启后，竟然忘了怎么开启ss的服务端。。。这也体现了记录的重要性，又折腾了得俩小时，泪的教训啊，这次虽然是第二次配，但也经历了很多弯路，现在来记录一下刚才的辛酸俩小时。。

ss一般配置在linux下，windows下的很少，网上教程也不多，但我的服务器是windows的，一直也没换，当时就想着在windows下试一试，所有才有了这篇文章。

首先，环境配置

Python(我用的版本是2.7.12)+pip+openSSL

注意：安装python和openssl的32位或者64位要统一

1.下载python和opensll（pip已经在安装python的过程中默认安装上了，在C:\Python27\Scripts\路径下）

2.环境变量配置，将C:\Python27\;C:\Python27\Scripts\放入path中

3.在cmd命令行中输入指令 pip install shadowsocks，显示“successfully installed shadowsocks”就是安装成功了

然后，保存配置文件

需要新建一个文本文档，改名为config.json，内容为：

{
“server”:”0.0.0.0”,
“server_port”:8388,
“local_address”:”127.0.0.1”,
“local_port”:1080,
“password”:”qwe123” //客户端连接服务器端需要的密码
“timeout”:300,
“method”:”aes-256-cfb”,
“fast_open”:false
}

保存到C:\Python27\Scripts\路径下

最后，开启服务

在cmd中输入指令 ssserver.exe -c C:\Python27\Scripts\config.json

现在服务器端已经配置结束

接下来看一看客户端的配置

首先下载ss客户端

然后按照刚才服务器端的配置，配置客户端

打开 启用系统代理和全局代理
